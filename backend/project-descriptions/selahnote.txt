PROJECT OVERVIEW
----------------
SelahNote (Lectra) is an AI-powered note-taking application for iOS that
transforms live recordings, uploaded audio, and YouTube content into structured,
editable notes through a hybrid local-first and cloud-synced architecture.
Built for students, church communities, professionals, and heavy content
consumers, SelahNote now goes beyond simple transcription and summarization by
adding authenticated cross-device sync, agentic workflow orchestration, live
WebSocket transcription, and vector-based scripture/reference intelligence.
Unlike traditional note-taking apps that stop at raw transcription or static
summaries, SelahNote runs an end-to-end intelligent workflow: capture audio,
stream speech in real time, persist content locally, sync it to the cloud,
generate structured sermon-style notes, detect references as the speaker talks,
and maintain the same organized workspace across devices. The result is a
real-time knowledge system that turns spoken content into structured, searchable,
reference-aware notes while preserving chronology and contextual meaning.
PURPOSE & CONTEXT
-----------------
The primary purpose is to serve as an intelligent audio-to-notes platform that
reduces the gap between listening and understanding. Users can record a sermon
or lecture live, upload an existing audio file, or paste a YouTube link, then
receive structured notes, scripture-aware summaries, and synced content that
remains available across sessions and devices.
The system addresses not only information overload, but also fragmentation.
Instead of trapping notes on one device or leaving users with long transcripts,
SelahNote transforms spoken content into synchronized, organized, and
intelligently enriched note objects. It answers questions like:
- "What were the main themes from this sermon?"
- "Which scriptures were quoted or referenced in real time?"
- "Can I revisit the exact point in the message when that verse came up?"
- "Will my folders, notes, and settings stay in sync across devices?"
SelahNote emphasizes evidence-first note generation and agentic enrichment:
structured notes are generated from the actual transcript, references are
detected from spoken content using semantic and explicit matching, and the full
workspace is hydrated from the cloud so users can continue where they left off.
TECH STACK
----------
- Frontend (iOS):
  * Framework: SwiftUI
  * Language: Swift 5.0+
  * Data Persistence: SwiftData
  * UI Architecture: MVVM with ObservableObject pattern
  * Audio Framework: AVFoundation (recording, playback, conversion, background audio)
  * Live Streaming Transport: Starscream WebSocket client
  * Networking: URLSession with async/await
  * Authentication: Firebase Auth with Sign in with Apple
  * Monetization: RevenueCat
  * Analytics: Firebase Analytics
  * Minimum iOS: 17.6+
- Backend Services:
  * Real-Time Transcription: AssemblyAI streaming WebSocket API
  * Batch Transcription: AssemblyAI pre-recorded transcription API
  * Structured Note Generation: OpenAI GPT-4o via Convex internal actions
  * Embeddings / Semantic Reference Detection: OpenAI embeddings API
  * Vector Search: Pinecone
  * App Backend / Sync Layer: Convex (HTTP actions, queries, mutations, internal actions)
  * YouTube Transcript Service: Google Cloud Function / Cloud Run Python service
  * YouTube Transcript Provider: Supadata API
  * Authentication Validation: Firebase token verification on backend endpoints
  * Cloud Infrastructure: Google Cloud + Convex
- Data Storage:
  * Local Persistence: SwiftData (on-device, local-first)
  * Cloud Persistence: Convex tables for users, folders, tuples, generation state, and references
  * Data Models: RootDirectory → Folders → TranscriptionTuples →
    (AudioFile, Transcription, LiveTranscription, Note, Structured Blocks, References)
  * Sync Metadata: remoteId, syncStatus, updatedAt, audioStorageRef, rtfStorageRef
  * File Format: Local audio stored as Data blobs (MPEG-4 AAC); structured notes and
    references stored as JSON for rendering and sync
- Audio Processing:
  * Recording Format: MPEG-4 AAC, 44.1kHz, mono
  * Live Streaming: WebSocket streaming to AssemblyAI using 16kHz PCM-16
  * Format Conversion: AVAudioConverter for real-time pipeline conversion
  * Turn Management: force-endpoint logic for long-form spoken turns
  * Background Processing: background task management for uninterrupted live capture
CODE ORGANIZATION
-----------------
- /Lectra:
  Main iOS application codebase. Contains SwiftUI views, SwiftData models,
  utilities, sync logic, and real-time audio intelligence modules.
  * /Views: SwiftUI views for folder navigation, recording, tuple detail,
    structured notes, reference timeline, Bible navigation, uploads, settings,
    onboarding, and the upcoming AI chat surface.
  * /Data Models: SwiftData models for RootDirectory, Folder, TranscriptionTuple,
    AudioFile, Transcription, LiveTranscription, Note, plus sync metadata and
    reference timeline block models.
  * /Utilities: Core business logic and service clients:
    - AudioRecorderManager: orchestrates recording, live transcription,
      structured note generation, and live reference detection
    - LiveTranscriptionController: AVAudioEngine capture + PCM conversion +
      streaming lifecycle control
    - AssemblyAIClient: batch transcription for uploaded/local audio
    - AssemblyAIStreamingClient / WebSocketManager: real-time AssemblyAI
      WebSocket transport and turn handling
    - ConvexSyncService: authenticated cloud sync, hydration, structured-note
      generation requests, and reference detection requests
    - FolderManager: local folder CRUD plus local-to-cloud bootstrap and remote hydration
    - YouTubeAudioDownloader: authenticated Cloud Run / Cloud Function transcript client
    - AuthManager: Firebase Auth + Sign in with Apple
    - CreditsManager: local/server credit state reconciliation
    - BibleStore / BibleService / ScriptureReferenceParser: scripture viewing and parsing utilities
  * /ViewModels: subscription, auth, and user plan state
  * /Theme: custom UI styling, note rendering, and app-wide design tokens
- /convex:
  Authenticated backend application layer. Defines schema, queries, mutations,
  HTTP routes, structured-note generation actions, and Pinecone-backed reference
  detection workflows.
- /AudioScraper:
  Python Google Cloud Function / Cloud Run service that validates YouTube URLs,
  verifies Firebase auth, consumes credits, fetches transcripts from Supadata,
  and returns normalized transcript payloads to the iOS client.
KEY FEATURES
------------
1. Real-Time Audio Recording with Live Transcription:
   - High-quality microphone capture with AVFoundation
   - WebSocket-based AssemblyAI streaming during recording
   - Partial and final transcript updates shown in real time
   - Turn-aware streaming pipeline with force-endpoint controls for long-form speech
   - Background task support, pause/resume, interruption handling, and route recovery
2. Agentic Real-Time Workflow:
   - Each finalized live speech turn can trigger backend reference detection
   - Server-side workflow decides whether the spoken content is a quote,
     reference, or "turn to" instruction
   - Timeline updates are merged back into the note object in real time
   - The app behaves like a live sermon/lecture copilot rather than a passive recorder
3. Multiple Audio Input Sources:
   - Direct microphone recording
   - Audio file upload from local files
   - YouTube transcript ingestion through authenticated cloud service
   - Fallback transcription pipeline for content without live transcript state
4. AI-Powered Structured Note Generation:
   - AssemblyAI provides the transcript layer
   - Convex internal actions call OpenAI to generate structured sermon notes
   - Output is schema-constrained JSON blocks rather than loose markdown
   - Supports headings, paragraphs, bullets, and scripture blocks
   - Generated titles can rename tuples automatically when appropriate
   - Notes remain editable and sync back to the backend
5. Cross-Device Sync:
   - Local SwiftData remains the immediate working store
   - Convex provides authenticated cloud sync for folders, tuples, notes,
     references, and user preferences
   - Local content can bootstrap into the cloud on first sync
   - Remote content hydrates back into SwiftData on launch and refresh
   - Sync state is tracked per tuple for more resilient cross-device behavior
6. Vector Search & Reference Intelligence:
   - OpenAI embeddings convert spoken transcript snippets into vectors
   - Pinecone is queried to detect semantic scripture matches
   - Explicit spoken references and semantic matches are both supported
   - Matches are classified as quote, reference, or turn-to events
   - References are timestamped and shown in a live chronological timeline
7. Reference Timeline Experience:
   - Scripture references detected during transcription are preserved as JSON blocks
   - Users can review quoted passages and jump into the in-app Bible view
   - Positive/negative review state allows feedback on detected references
   - Timeline entries are merged intelligently to reduce duplicate detections
8. YouTube Integration:
   - Supports normalized YouTube URL validation
   - Authenticated cloud service fetches transcript content from Supadata
   - Credit consumption is enforced on the backend
   - Returned transcripts can flow directly into structured-note generation
   - Tuple metadata preserves the original YouTube link
9. Authentication, Billing, and User State:
   - Firebase Authentication with Sign in with Apple
   - RevenueCat subscription and entitlement resolution
   - Backend-authenticated endpoints protected by Firebase token verification
   - Server-backed credit state for usage-based flows
   - User preferences such as language, appearance, and church reminder settings sync to Convex
ARCHITECTURE FLOW
-----------------
1. Recording Workflow:
   User starts recording → AVAudioEngine captures audio → LiveTranscriptionController
   converts audio to PCM-16 → WebSocket streams to AssemblyAI → Partial/final turns
   update the live transcript → Finalized turns trigger Convex reference detection →
   detected scripture/reference blocks merge into the local timeline → recording stops →
   transcript persists locally and syncs to Convex → structured notes are generated via
   Convex + OpenAI → results sync back into the tuple
2. Upload Workflow:
   User selects audio file → Audio is saved locally → TranscriptionTuple created →
   AssemblyAI batch transcription runs if needed → transcript persists locally →
   Convex structured-note generation action produces block-based notes → tuple syncs
   across devices
3. YouTube Workflow:
   User pastes YouTube URL → URL validated → Firebase-authenticated cloud request sent →
   backend verifies token, consumes credit, and fetches transcript from Supadata →
   transcript returns to iOS → tuple created locally → tuple syncs to Convex →
   structured notes generated and saved
4. Note Generation Pipeline:
   Live transcript or batch transcript → Convex internal action loads owned tuple →
   OpenAI GPT-4o generates schema-constrained structured notes → title extracted from
   h1 when applicable → structuredBlocksJson saved to tuple → editable structured UI
   renders in TupleView → changes sync back to Convex
5. Data Persistence Flow:
   SwiftData stores the working set locally → FolderManager hydrates from Convex on launch →
   local mutations mark tuples/folders dirty → ConvexSyncService upserts authenticated data →
   Convex becomes the shared cross-device state → refreshed data rehydrates local models while
   preserving responsive on-device UX
KEY INNOVATION: DUAL TRANSCRIPTION MODALITIES
---------------------------------------------
SelahNote now combines dual transcription with agentic post-processing:
1. **Live Streaming Transcription**: During recording, audio is streamed in
   real time via WebSocket to AssemblyAI, giving the user immediate transcript
   feedback and enabling downstream real-time enrichment such as reference
   detection.
2. **Batch / Deferred Transcription**: For uploaded files or recordings that
   need a fallback pass, AssemblyAI batch transcription provides a robust
   transcript source for structured-note generation.
The major evolution is that transcription is no longer the end of the pipeline.
Each transcript can feed an agentic backend workflow that classifies references,
queries vectors, generates structured notes, and synchronizes the result across
devices.
NOTE GENERATION: CUSTOM PROMPT ENGINEERING
------------------------------------------
The note generation system now uses a stricter structured-output workflow
designed for sermon-style notes:
- **Structured Output**: OpenAI returns validated JSON blocks instead of freeform markdown
- **Semantic Types**: Blocks are constrained to heading, paragraph, bullet, and scripture
- **Hierarchy Enforcement**: h1, h2, and h3 levels preserve a clean sermon outline
- **Scripture Metadata**: Scripture blocks carry reference and translation metadata
- **Chronological Grounding**: Notes are generated from actual transcript content
- **Editable Results**: Structured notes can be rendered and edited in the iOS UI
- **Language Awareness**: The generation request respects the user-selected transcription language
This turns note generation into a predictable, renderable, and sync-friendly
content pipeline rather than a one-off markdown response.
DATA MODEL ARCHITECTURE
------------------------
The app uses SwiftData locally and Convex remotely, with the tuple acting as the
central synchronized content object:
- **RootDirectory**: Top-level local container
  - Contains: Folders array
- **Folder**: Organizational unit
  - Contains: TranscriptionTuples array
  - Syncs to Convex using stable client-side UUIDs
- **TranscriptionTuple**: Core synchronized entity
  - Optional AudioFile
  - Optional Transcription
  - Optional LiveTranscription
  - Optional Note
  - Structured note payload via `structuredBlocksJson`
  - Reference timeline payload via `referencesJson`
  - Sync metadata: `remoteId`, `syncStatus`, `updatedAt`
  - Remote asset metadata: `audioStorageRef`, `rtfStorageRef`
  - User metadata: `createdAt`, `lastOpenedAt`, `youtubeLink`
- **User Preferences**:
  - Stored locally in UserDefaults/AppStorage
  - Synced to Convex for cross-device continuity
  - Includes onboarding responses, church reminder timing, language,
    appearance, and notification state
This architecture supports a local-first UX while enabling authenticated cloud
hydration, durable sync, and structured AI enrichment.
CLOUD INFRASTRUCTURE: YOUTUBE TRANSCRIPT SERVICE
------------------------------------------------
The YouTube transcript service now acts as an authenticated backend edge layer
rather than a simple proxy:
- Accepts authenticated requests with Firebase bearer tokens
- Validates and normalizes YouTube URLs
- Enforces server-side credit consumption before transcript retrieval
- Calls Supadata to fetch transcript content
- Returns transcript payload plus server credit state
- Supports a credits lookup endpoint for app-side reconciliation
- Runs as a Python Google Cloud Function / Cloud Run-style deployment
This service offloads transcript retrieval, credit enforcement, and auth-aware
request handling from the iOS app while keeping the client lightweight.

PROJECT POSITIONING
-------------------
SelahNote is positioned as a premium AI-powered iOS knowledge system for spoken
content, not just a transcription app. Its differentiation comes from combining
real-time audio capture, agentic backend workflows, vector-based scripture
detection, structured note generation, and authenticated cross-device sync into
one cohesive experience.
The platform emphasizes:
- **Real-Time Intelligence**: live WebSocket transcription and turn-aware enrichment
- **Agentic Workflow Design**: backend actions coordinate note generation and reference detection
- **Cross-Device Continuity**: Convex sync keeps folders, tuples, and preferences aligned
- **Vector-Aware Retrieval**: Pinecone enables semantic scripture/reference matching
- **Structured Output**: notes are generated as renderable and editable JSON blocks
- **Flexibility**: recording, file upload, and YouTube ingestion support multiple content paths
- **Production Readiness**: Firebase auth, RevenueCat billing, server credit enforcement,
  and cloud-backed APIs support real app usage
The addition of Convex, Pinecone, OpenAI embeddings, authenticated backend
actions, and real-time reference intelligence moves SelahNote from a
transcription-and-summary app into a more agentic, synchronized AI note platform
built for live interaction, structured understanding, and long-term knowledge
retention.
